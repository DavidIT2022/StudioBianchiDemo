Capitolo 01

Il Caso Studio Bianchi

Uno studio professionale che offre servizi di consulenza contabile, fiscale e revisione a PMI e grandi aziende.

12 Professionisti

Team organizzato in 3 aree: Bilanci e Contabilità, Fiscale, Revisione. Ogni persona ha storico di ruolo, costo e team.

250 Incarichi

Incarichi con 180 clienti diversi. Ogni incarico ha un importo pattuito, un capitolato e ore lavorate.

16.248 Ore Registrate

Due anni di timesheet (2024-2025) con dettaglio giornaliero delle ore per persona e incarico.

Analisi Marginalità

Confronto tra ricavo reale (pattuito) e ricavo teorico (ore × tariffa). Calcolo margine effettivo vs desiderato.

Capitolo 02

Modello Star Schema

Architettura dimensionale secondo la metodologia Kimball con SCD Type 2 per la storicizzazione.

┌─────────────────┐ │ Dim_Data │ │ (DataKey) │ └────────┬────────┘ │ ┌────────────────────────────┼────────────────────────┐ │ │ │ ▼ ▼ ▼ ┌──────────────┐ ┌─────────────────┐ ┌────────────────┐ │Fact_Timesheet│ │Fact_Redditivita │ │Fact_CostiDir. │ │ (dettaglio) │ │ _Incarico │ │ (spese vive) │ │ │ │ (snapshot) │ │ │ │- OreLavorate │ │ │ │- Importo │ │- CostoEffect.│ │- RicavoReale │ │ │ │- RicavoTeor. │ │- RicavoTeorico │ └───────┬────────┘ │- MargineTeor.│ │- MargineReale │ │ └──────┬───────┘ │- MargineDeside. │ │ │ └────────┬────────┘ │ │ │ │ ├────────►Dim_Persona◄──────┤ │ │ (SCD2) │ │ │ │ │ └─────────►Dim_Incarico◄────┴───────────────────────┘

Surrogate Keys

DataKey (YYYYMMDD) e PersonaSK per join ottimizzati e gestione SCD.

SCD Type 2

Dim_Persona storicizza ruolo, costo orario, tariffa e team con ValidoDa/ValidoA.

Role-Playing

Dim_Team usata sia come TeamPrestatore che TeamTitolare nella Fact_Timesheet.

Denormalizzazione

Dim_Incarico include Cliente e TipoIncarico embedded per ridurre join.
